\section{Introduction}

In this Thermal Systems project, we set out to design a fin bank for a given duct,  create a model describing the system as a whole, and analyze how changing different variables within the model affects the system. The geometry of the duct itself was given to us, and a simplified version of the geometry can be found in Figure XXX in the appendix. To simplify the task, this project was split into three parts, described below.

\subsection{Part 1 - Fin Bank}

The first part of this project focused on developing the model used to describe the fin bank. While there were many ways to model the system (iterative solver software, Python, etc.) we chose to use MATLAB because of it's familiarity and ease of use with GitHub for sharing among teammates. This part was the most straightforward of the project, simply consisting of calculating various parameters required for describing a fin bank. This included fin efficiencies, fin bank areas, and different resistances. After creating our model, we verified it by using it to solve Example 2.11(a) from Slide 79 of the Thermal Conduction notes, previously solved in class by Dr. Miller. This script, along with all other MATLAB scripts used throughout the project, can be found in Figures XXX-XXX in the appendix.

\subsection{Part 2 - Cooling System}

The second part of this project focused on modifying our model to account for the cooling system, i.e. the air flow through the duct from a cooling fan. This included drawing a diagram of the "ducts" between each fin, which we treated as an enclosed duct. This diagram can be found in Figure XXX in the appendix. Additionally, this part of the project was the most computationally complex, as we needed to compute the hydraulic diameter and Reynolds number for the flow between fins, estimate the bulk temperature and mean air velocity in the duct, the convective heat transfer coefficient of the flow between fins, and the fan power required to cool the system.

\subsection{Part 3 - Performance Analysis}

The third and final part of this project sought to use the models developed in the first two parts and evaluate their performances as a function of various design parameters for the fin bank (number of fins, fin thickness, fin length, etc.). The problem statement provided a defined fin bank geometry, which we were able to use to analyze the model's performance.  Additionally, the performance was computed used a given benchmark CPU performance score given by the problem statement.

\section{Assumptions}

In order to properly complete this project, we had to make some assumptions along the way. The assumptions we made are as follows:
\\

\begin{quote}
1. Assume all heat from CPUs is dissipated through the fin bank. This greatly reduced the amount of work and considerations needed for later calculations. 

2. Assume 1-D heat transfer through the entire system (perpendicular to the duct wall and perpendicular . This is a typical assumption for fin bank analysis. 

3. Assume steady state conduction and fin convention. These are typical assumptions for fin bank analysis.

4. Assume that, for a given side, there is a space between the first fin and the wall, and a space between the last fin and the first fin on the adjacent sides. This reduces work by preventing extra conduction between touching fins.

5. Assume each fin has an identical profile. This reduces work by allowing us to calculate the heat transfer through one fin and simply multiply by the number of fins.

\end{quote}
